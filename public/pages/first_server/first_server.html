Code conventions / Clean Code (Strict Mode - ESLint)

Som nævnt tidligere, så er kode konventionerne og Clean Code principperne afgørende for at ens kode er læsbar, effektiv og nemt at vedligeholde. Hvis man gør brug af Strict Mode og værktøjer som ESLint, vil det være nemmere at overholde disse standarder.

Strict Mode

"use strict"; er en funktion i Javascript, der skal skrives i toppen af ens fil eller funktion. Når man bruger denne funktion, bliver ens kode underlagt ekstra restriktioner, som kan hjælpe med at fange fejl, der ellers kunne være overset. Det betyder, at man kan opdage potentielle problemer tidligere i udviklingsprocessen. Det gør ens kode mere pålidelig og nemmere at fejlsøge. Ved at bruge "use strict" kan man skrive renere og mere sikker kode, hvilket vil give bedre resultater i sidste ende.

ESLint

ESLint er et værktøj til der hjælper med at finde fejl og rette problemer i ens Javascript kode. Ved at bruge ESLint, så vil man som udvikler sikre, at ens kode overholder kode konventionerne og Clean Code principperne. ESLint er konfigurerbar, hvilket betyder udvikler teams selv kan definere deres egne regelsæt baseret på deres specifikke krav og behov. Den mest almindelig fejl og som jeg selv ofte oplever, er at man nogle gange overser, at der sættes semikolon til sidst. Det kan nogle gange være meget frustrerende, derfor er det et super godt værktøj til at identificere fejl og selv de små fejl.

Fordelene ved at bruge dette værktøj er:

  • Man kan forbedre sin kvalitet af kode, som hjælper med at identificere mønstre, der kan føre til fejl.
  • Der er konsistens, når der bliver skrevet noget kode af forskellige udviklere og det sørger for at man følger de regelsæt og standarder som er blevet defineret.
  • Det vil effektivisere processen, fordi det automatisk hjælper en med at finde og rette almindelige fejl, som resulterer i, at man spare tid under kodegennemgang og fejlfinding.

Variables and Functions

I JavaScript er det essentielt at forstå, hvordan variabler og funktioner bliver tilgængelige i ens kode. Scope er det, der bestemmer, hvor i koden variabler og funktioner kan bruges.

Der findes primært tre typer af scopes.

  • Global scope
  • Function scope
  • Block scope

Global scope

Når man definerer en variabel uden for en funktion, så bliver den en del af det globale område. Det betyder, at den kan tilgås og ændres fra alle steder i ens kode.

Eksempel

  
    var globalVar = "Jeg kan tilgås i global scopes";

    function showGlobal() {

        console.log(globalVar); 

    }

    showGlobal(); 

    console.log(globalVar);

Function scope

Når man erklærer variabler i en funktion, er de kun tilgængelige inden for den pågældende funktion. Dette kaldes "function scope". 

  
    function showLocal() {

        var localVar = "Jeg er lokal og kan kun tilgås inde i
    funktionen";

        console.log(localVar);

    }

    showLocal();

    console.log(localVar);

Block Scope

Når man bruger let eller const til at deklarere variabler inden for en blok {} - f.eks. i en if-sætning eller en for-løkke - så er de kun tilgængelige inden for den pågældende blok. 

  
    if (true) {

        let blockVar = "Jeg er blok-scoped";

        console.log(blockVar); 

    }

Functions

Når du skriver en funktion, skal du først vælge et navn til den et sæt af parametre, og skrive den kode, der skal udføres, når funktionen bliver brugt.

Eksempel

  
    function hello() {

        return "Hello, World!";

    }

    

    console.log(hello());

Callback functions

Callback funktioner bruges som argumenter til andre funktioner og kan eksekveres på et senere tidspunkt. Dette gør det muligt at udføre handlinger asynkront, for eksempel håndtere begivenheder eller tilpasse funktionalitet efter behov.

Eksempel

  
    function generiskHandling(navn, handling) {

        return handling(navn);

    }

    

    // Her bliver der defineret en række callback-funktioner

    const svømmer = (navn) => `${navn} svømmer.`;

    const læser = (navn) => `${navn} læser en bog.`;

    const sover = (navn) => `${navn} sover.`;

    

    // Udfører generiskHandling med forskellige callbacks

    const svømmerResultat = generiskHandling("Dennis", svømmer);

    const læserResultat = generiskHandling("Dennis", læser);

    const soverResultat = generiskHandling("Dennis", sover);

    

    console.log(svømmerResultat); // Output: "Dennis svømmer."

    console.log(læserResultat); // Output: "Dennis læser en bog."

    console.log(soverResultat); // Output: "Dennis sover."

    

    // Bruger en inline callback-funktion direkte som argument

    console.log(generiskHandling("Dennis", (navn) => `${navn} spiser.`));
    

    // Output: "Dennis spiser."

package.json

HVad er package.json

package.json er en essentiel fil i vores Node.js projekt som hjælper os med at holde styr på hvilke programmeringspakker(dependencies) vores projekt gør brug af og sikrer, at alle bruger de samme version for at undgå problemer. Filen indeholder også generelle informationer, såsom projektets navn, ejerskab og beskrivelse osv. Man kan også definere forskellige kommandoer for at køre opgaver som at teste eller bygge projekter.

Det vil sige, at en package.json fil er rigtig brugbar for os udvikler, da det er med til at hjælpe med at organisere og gør det super let for os at samarbejde om et projekt.

Hvordan opretter man en package.json fil?

For at oprette en package.json fil skal man først sørge for at have Node.js installeret:

1. Naviger hen til projektmappen

Åbn en terminal og naviger hen til ens projektmappe. Her kan du gør brug af kommandoen cd for at skifte hen til mappen, hvor dit projekt er.

2. Kør kommandoen

Efterfølgende skal du køre kommandoen npm init for at starte processen med at oprette din package.json fil. Når man kører denne kommando, så vil npm stille nogle spørgsmål omkring dit projekt. Det er f.eks. projektets navn, version, beskrivelse osv. Hvis du derimod vil springe spørgsmålene over og gå direkte over til oprette en default fil, så skal du bruge kommandoen npm init -y

3. Installere forskellige dependencies

Vi har gjort brug af express i undervisningen og så når vi har installeret express til vores projekt ved at bruge kommandoen npm install express så vil ens package.json fil bliver opdateret og det vil inkludere Express som en afhængighed/dependency og vil samtidig også oprettet ene node_modules mappe til ens projekt, hvor Express og dets afhængigheder vil blive installeret.

4. Klar til at tilpasse efter projekts behov

Når du har fået oprettet din package.json har du mulighed for at tilpasse det efter dit behov. Nu kan du manuelt tilføje scripts eller andre afhængigheder i din package.json fil.

Eksempel på en default package.json fil

  
    {

      "name": "mit-projekt",

      "version": "1.0.0",

      "description": "En kort beskrivelse af mit projekt",

      "main": "index.js",

      "scripts": {

        "start": "node index.js"

      },

      "author": "Dit Navn",

      "license": "ISC"

    }

Express

Express er et fantastisk redskab til at bygge webservere og API'er med Node.js. Det er kendt for sin hurtighed, enkelhed og skalerbarhed, så du kan fokusere på at kode frem for at bekymre dig om tekniske detaljer. Med Express kan du hurtigt opsætte middleware til at håndtere HTTP-forespørgsler, definere ruter baseret på URL'er og HTTP-metoder, og interagere effektivt med databaser. Det gør udviklingen af web- og mobilapplikationer enklere og mere strømlinet.

Installation af Express

Når du har oprettet din projektmappe og initialiseret et nyt node.js projekt. Så skal du bruge kommandoen npm install express og det vil nu automatisk opdatere din package.json og package-lock.json. Det vil også oprette en node_modules mappe, hvis den ikke allerede eksisterer, hvor Express og dets afhængige pakker vil blive installeret.

Fordele ved at bruge Express

Simpel og enkelt

Når det kommer til bygning af webapplikationer og API'er med Node.js, gør Express det virkelig nemt. Det handler om enkelhed. Du tilføjer kun det, du rent faktisk har brug for, hvilket betyder, at din applikation forbliver let og effektiv.

Fleksibelt

Express er virkelig fleksibel på grund af sin enkelhed. Du kan strukturere din applikation på præcis den måde, du vil. Du er ikke låst fast til en bestemt tilgang, så du kan bruge lige præcis de biblioteker, som du bedst kan lide og passer til dit behov.

Middelware

Middleware i Express fungerer ved at tillade asynkrone handlinger at udføres i en bestemt rækkefølge, hvilket giver dig kontrol over håndtering af forespørgsler og svar. Det er nyttigt til opgaver som at analysere indgående anmodningsdata, administrere cookies og implementere brugerautentificering.

Routing

Med Express kan vi som udviklere nemt definere ruter til at håndtere forskellige HTTP-anmodninger med deres kraftfulde routing API. Dette gør det muligt at opbygge både RESTful web tjenester og applikationer, der følger de seneste webstandarder.

Dokumentation og support

Ekspress leveres med en grundig vejledning, som er let at følge, hvilket gør det tilgængeligt for både nybegyndere og erfarne udviklere. Udover dette er der solid support fra fællesskabet gennem platforme som Stack Overflow og GitHub, hvilket effektivt hjælper med at løse problemer i en fart.

Sending data with GET request

Sending af data med GET-forespørgsler er en almindelig praksis i webudvikling, især når det kommer til at anmode om data fra en server. GET-forespørgsler er en af de HTTP-metoder, der anvendes til at anmode om data fra en specifik ressource på internettet. Når du sender data med en GET-forespørgsel, inkluderes dataene i selve URL'en, typisk i form af query-strenge.

Sådan fungerer det

URL med Query-strenge

Data sendes som en del af URL'en i form af query-strenge. En query-streng starter efter et spørgsmålstegn (?) og består af nøgle-værdi-par, hvor hver nøgle og værdi er adskilt af et lighedstegn (=), og hvert par er adskilt af et ampersand (&). For eksempel: http://example.com/api/users?name=John&age=30.

Ingen kropsindhold

I modsætning til POST-forespørgsler, som sender data i kroppen af forespørgslen, har GET-forespørgsler ikke en krop. Dette betyder, at al data, der skal sendes, skal inkluderes i URL'en.

Hvordan kan det bruges

Dataanmodninger

GET-forespørgsler bruges typisk til at anmode om data baseret på parametre angivet af brugeren. For eksempel kan en søgeside bruge GET-forespørgsler til at anmode om søgeresultater baseret på søgeord angivet af brugeren.

Hente specifikke ressourcer

Når du kender identifikatoren for en specifik ressource, kan du bruge en GET-forespørgsel med den pågældende identifikator for at hente den ressource. For eksempel, anmodning om en specifik brugerprofil baseret på brugerens ID.
