Intro til Node.js

Hvad er Node.js?

Node.js er et gratis open-source JavaScript-runtime-miljø, der lader os udviklere skabe servere, webapplikationer, kommandolinjeværktøjer og scripts som kan køre på forskellige platforme såsom Windows, Linux, Mac OS X m.m.

Hvordan installerer man det?

Trin 1: Download Node.js

Du skal starte med at gå ind på https://nodejs.org/en og vælge den version, der passer til dit operativsystem. Det anbefales at vælge LTS-versionen (Long Term Support), da den er mest stabil og testet med npm. Derefter følger du bare anvisningerne, og så er du klar til næste skridt.

Trin 2: Kontrol af version

Efter du har gennemført installationen, så er det en rigtig god ide at kontrollere din version af Node.js og npm: Når Node.js er installeret, kan du kontrollere din version ved at køre node -v (viser hvilken version af node du har installeret) og npm -v (viser hvilken version af npm du har installeret) i din terminal eller kommandolinje. Når du har kontrolleret versionen og det hele er korrekt, så kan du komme i gang med dit første projekt.

Fordele ved Node.js

Enkel syntaks

Node.js har en enkel og intuitiv syntaks, hvilket gør det nemmere at lære og bruge, især hvis man har godt styr på Javascript.

Mulighed for at skalere hurtigt

Node.js er asynkron, som betyder at systemet kan håndtere flere forespørgsler på én gang. Det vil sige, at systemet ikke venter på, at Input/Output-operationen er færdiggjort, før en ny anmodning kan behandles. Det er event-drevet hvilket gør det ideelt at udvikle skalerbare webapplikationer, der bruger langt færre ressourcer.

Det er open-source og fleksibel

Node.js er open-source, som betyder, at det er gratis og super fleksibel, fordi det kan tilpasses afhængig hvilket projekt man gerne vil udvikle.

Variabler og datatyper

Variabler

Variabler i JavaScript er brugt til at gemme og håndtere data. De er dynamiske, hvilket betyder, at de kan holde forskellige datatyper over tid. Javascript understøtter otte datatyper: String, Number, BigInt, Boolean, Undefined, Null, Symbol, og Object. Der kommer en lille forklaring på datatyper nedenfor. Variabler kan oprettes ved hjælp af var, let, eller const. I dette fag bruger const som default f.eks const myName = "Dennis";

const er konstant og ikke i værdien men i tildelingen, hvilket betyder, at den ikke kan omplaceres og derfor også skal være tildelt. Vi skal til enhver tid, hvis det er muligt bruge const og const kan ikke være uangivet eller genangivet, men dens værdi kan ændres, så længe du ikke ændrer opgaven.

Det man aldrig må gøre, når man definerer en variabel.

totalGlobalVariable = "Never EVER!!! do this!!!"; 

 var globalVariable = "Also NEVER ever do this!!!";

Datatyper

I JavaScript er variabler dynamiske, hvilket betyder, at samme variabel kan bruges til at holde forskellige datatyper. Dette gør JavaScript fleksibelt og giver os udviklere mulighed for at ændre datatypen af en variabel efter behov

JavaScript har otte datatyper:

  • String: Tekststrenge
          
        let tekst = "Hello World!";
    
        console.log(tekst); // Output: "Hello World!"
  • Number: Tal, både heltal og decimaler
          
        let heltal = 42;
    
        let decimaltal = 3.14;
    
        console.log(heltal, decimaltal); // Output: 42 3.14
  • Boolean: Sand eller falsk
          
        let sandt = true;
    
        let falsk = false;
    
        console.log(sandt, falsk); // Output: true false
  • Undefined: Variabler uden en værdi har typen undefined
          
        let undefined;
    
        console.log(undefined); // Output: undefined
  • Null: Repræsenterer en tom eller ikke-eksisterende værdi
          
        let tom = null;
    
        console.log(tom); // Output: null
  • Symbol: Unikke og uforudsigelige værdier
          
        let symbol1 = Symbol("a");
    
        let symbol2 = Symbol("a");
    
        console.log(symbol1 === symbol2); // Output: false
    
        // Selvom vi gav dem samme navn, er de forskellige
  • Object: Samlinger af nøgle-værdi par
          
        let person = {
    
        navn: "Dennis",
    
        alder: 28
    
        };
    
        console.log(person); // Output: { navn: "Dennis", alder: 28 }
  • BigInt: Større tal, der kan være større end det maksimale sæt af tal, der kan repræsenteres af Number 
          
        let stortTal = BigInt(9007199254740991); // Eller 9007199254740991n
    
        console.log(stortTal); // Output: 9007199254740991n

REST API Konventioner

REST API konventioner og bedste praksis omhandler, hvordan man designer og implementerer API'er på en måde, der er intuitiv og let at bruge. Disse principper sikrer, at API'er bliver konsistente, skalerbare og sikre, hvilket gør det nemmere for udviklere at integrere og vedligeholde dem over tid.

Nogle af de vigtigste konventioner:

  • Gør brug af nøgler i stedet for verber i Endpoints. Det vil sige, man bør bruge nøgler, der repræsenterer ressourcer. HTTP-metoder som GET, POST, PUT, PATCH og DELETE skal bruges til at udføre handlinger på disse ressourcer, hvilket effektivt gør dem til verber. Dette gør API'en intuitiv og let at forstå.
  • Undgå at bruge verber i URI'er, som nævnt ovenfor, så vil det være mest passende at bruge nøgler til at repræsentere ressourcer, og det vil ikke være hensigtsmæssigt at inkludere verber i REST URI'er. HTTP-metoder (GET, POST, PUT, DELETE, osv.) bruges til at udføre handlinger på disse ressourcer. Hvis vi bruger verber i URI'en, skaber vi højst sandsynlig en RPC-stil metodeopkald med et JSON eller XML anmodnings-/svarsformat, hvilket ville være forkert at kalde REST.
  • Klare Versionering af REST API’er, så vil du på denne måde ikke tvinge klienter til at migrere til de nye versioner, hvis de vælger ikke at gøre det. Dette gør det muligt for klienter at bruge den version, de er mest komfortable med, uden at risikere at bryde applikationen.

Kode konventioner / Clean code

Kode konventioner og Clean Code principper er retningslinjer og bedste praksis for at skrive kode, der ikke kun fungerer effektivt men også at koden er nemt at læse, forstå og arbejde med, både for dig selv og andre udviklere.

Ved at følge kodekonventioner og Clean Code principperne, vil det hjælpe med at reducere fejl, hvilket gør det lettere at debugge ens kode for fejl.
